# 黑卡闪问题输入信息规范

> 版本 v1.0 | 2026-05-27  
> 用途：确保每个 Bug 报告包含足够信息，让分类引擎能准确归类

---

## 为什么需要规范

当前 673 条 Bug 中，32 条无法分类（"需人工判断"）——**100% 都是因为没有填写 Comments**。

```
分类准确率 vs 信息完整度
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Comments 有内容   → 分类准确率 >95%
Comments 空白     → 分类准确率 = 0%  ← 全部落入"需人工判断"
```

---

## 字段分级

### 🔴 必填（缺一不可）

| # | Excel 列名 | 填写要求 | 为什么 |
|---|-----------|---------|--------|
| 1 | **Bug ID** | 系统自动生成 | 唯一标识 |
| 2 | **Title** | 必须包含**技术关键词**（模块/现象/触发条件） | 关键词匹配第一入口 |
| 3 | **Comments** | 分析过程、日志片段、复现条件、怀疑方向 | **32条无法分类全因这个空白** |
| 4 | **Cause Analysis** | 具体根因描述，不能只是"状态异常" | 决定分类细粒度的关键 |

### 🟡 强烈建议

| # | Excel 列名 | 填写要求 | 为什么 |
|---|-----------|---------|--------|
| 5 | **Solved Scheme** | 修复方式/对策 | 修复方式可反推根因类别 |
| 6 | **Module** | 所属模块（底软/System UI/多媒体等） | 缩小分类匹配范围 |
| 7 | **Frequency** | 必现/偶发/概率X% | 区分环境问题 vs 系统缺陷 |
| 8 | **Status** | Closed/Analysis/Confirm/Open | 增量过滤依赖此字段 |

### 🟢 锦上添花

| # | Excel 列名 | 用途 |
|---|-----------|------|
| 9 | **Severity** | A/B/C 分级，A级优先关注 |
| 10 | **Sample Stage** | 样件阶段，区分开发期 vs SOP |
| 11 | **Assignee** | 责任人 |
| 12 | **Analyzer** | 分析人 |
| 13 | **Actual Result** | 日志路径（工作流二使用，不参与分类） |

## 各字段填写示例

### Title（标题）规范

```
❌ 太模糊:
  "黑屏"
  "卡顿"
  "显示异常"

✅ 包含技术关键词:
  "【STR】休眠唤醒2528次后中控黑屏，仪表正常"
  "【CarPlay】连接CP切歌时中控黑屏5秒后恢复"
  "【AVM】挂R档后倒车影像延迟3秒出现"
  
格式: 【模块/场景】现象描述 + 关键条件
```

### Comments（备注/分析说明）规范

```
❌ 空白或不规范:
  (空)
  "已分析"
  "同上"

✅ 结构化填写:
  复现条件: 连接CarPlay后打开分屏，猎鹰界面黑屏
  日志分析: QNX sail日志显示surfaceflinger进程冻结
  怀疑方向: CarPlay分屏模式下surface渲染异常
  关联Bug: BUG20260407_03660 (同类问题)
  
格式: 复现条件+日志片段+怀疑方向+关联问题
```

### Cause Analysis（根因分析）规范

```
❌ 无效:
  "根本原因暂不确定"
  "状态异常"
  "需要进一步分析"
  (直接复制标题)

✅ 具体:
  "分屏切换时CarPlay的Surface未正确detach，
   导致surfaceflinger合成超时→冻屏→黑屏"
   
  "str_svp_start.sh调用env_setup.sh时路径变更，
  svp应用启动失败导致STR退出后黑屏"
  
格式: 直接原因 → 中间环节 → 最终现象（因果链）
```

### Solved Scheme（解决对策）规范

```
✅ 具体:
  "分屏切换时增加surface状态检查，延迟200ms释放"
  "修改env_setup.sh路径引用，改为相对路径"
  "横展IPCL密钥同步补丁策略，防止开机反复同步"
```

## 工作流一分类依赖关系

```
Title ──────────┐
Comments ───────┤
Cause Analysis ─┤──→ _build_analysis_text() ──→ _classify_weighted()
Solved Scheme ──┘         │                          │
                           │                    28类规则匹配
Module ────────────────────┘                    strong=3分
Frequency ─────────────────┘                   medium=2分
                                               weak=1分
Status ──→ _classify_fix_status()
            已修复/未修复/无法复现/分析中/待确认

Actual Result ──→ 工作流二（日志下载+分析）
                  不参与分类！
```
